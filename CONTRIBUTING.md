# Contributing to Koluman Games 🎮

First off, thank you for considering contributing to Koluman Games! It's people like you that make this platform better.

## 🚀 How Can I Contribute?

### Reporting Bugs
- Use the **Bug Report** template when opening an issue.
- Describe the bug in detail and provide steps to reproduce it.
- Include screenshots or screen recordings if possible.

### Suggesting Enhancements
- Use the **Feature Request** template when opening an issue.
- Explain why this enhancement would be useful and how it should work.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes (`npm run build`).
4. Format your code according to the project's style (ESLint).
5. Open a Pull Request with a clear title and description.

## 🌿 Branching Strategy
- `main`: Production-ready code. No direct commits allowed.
- `feat/feature-name`: For new features.
- `fix/bug-name`: For bug fixes.
- `docs/changes`: For documentation updates.

## 🎨 Code Style
- We use **TypeScript** for type safety.
- **Vanilla CSS** with modern custom properties for styling.
- Follow the existing folder structure (`src/components`, `src/assets`, etc.).
- Use descriptive variable and function names.

## 🌍 Internationalization (i18n)
If you add new text to the UI, ensure it's added to all translation files in `src/i18n.ts`:
- English (en)
- Turkish (tr)
- Swedish (sv)
- German (de)
